Privacy Policy for fyle

Version: 1.0
Effective Date: 05 April 2025
Last Updated: 05 April 2025

Introduction

fyle is a utility app designed to assist users in managing documents such as insurance files, academic certificates, vehicle-related papers, and more. It allows users to upload documents, extract and save key information, categorise them, and set reminders for important dates. The App functions entirely offline and stores all data locally on the user's device.

We do not collect personally identifiable information unless provided through document content, and no user login or online service is required to use the App.

Data Collection and Usage

Information Collected

The App may collect and store the following types of data strictly on-device:

  • Uploaded files (images, PDFs)
  • Extracted or user-entered metadata such as:
    • Document title
    • Expiry or renewal date
    • Summary or description
    • Category tag
    • Reminder date
    • Favorite status

This information is either manually entered by the user or derived through on-device machine learning and optical character recognition (OCR). All data remains strictly on the device.

No User Account or Authentication

fyle does not require users to sign in or register. The App does not collect contact details such as name, email address, or phone number.

Permissions Used

The App may request access to:

  • Camera - to scan physical documents.
  • Photos - to allow importing document images.
  • Files - only if the user explicitly imports files using iOS' native file picker.
  • Local Network - used for document sharing via Multipeer Connectivity.
  • Local In-App Notifications - used to remind users of upcoming document expiry or renewal dates.

All permissions are requested with the user's consent, and the app functions in a limited capacity if permissions are declined. None of this data is transmitted off-device or accessed without the user's permission.

Data Not Collected

The App does not access or collect:

  • Contacts
  • Location data
  • Health data
  • Audio (microphone)
  • Biometric data
  • Advertising identifiers
  • Analytics or crash reporting metrics
  • Financial Data
  • IP Address or Device Identifiers
  • Browsing History & Search History

Data Storage and Security

All data is stored locally on the device using Apple's Core Data framework. There is no remote database or server backend, and we do not have access to user data.

Sharing Data

Users may share documents using:

  • System share sheet - standard PDF or file export through iOS
  • Multipeer Connectivity - local peer-to-peer transfer to another fyle user

When documents are shared, the data is stored only on the recipient's device and is not transmitted to external servers. We do not track, log, or process these transfers.

Encryption and App Security

At present, fyle does not encrypt data at rest. However, we recommend users use device-level security (passcode, Touch ID, or Face ID) to protect their data.

Future updates may include:

  • Local encryption of stored files
  • App-level lock features

Data Retention

All data remains on the user's device for as long as the app is installed. Once the app is deleted from the device, all user data, including documents, reminders, and extracted information, is permanently removed and cannot be recovered.

Data Sharing and Disclosure

We do not sell, rent, or share any user data with third parties.

We do not use any analytics, advertising SDKs, or third-party APIs that process personal data.

All features operate offline, and there is no centralized server or internet-based data storage involved.

User Controls and Choices

Deletion of Data

Users may delete any document or reminder at any time. Deleted data is permanently removed from the device and cannot be recovered.

Export of Data

Users may export individual files or share entire documents through the system share options or via Multipeer Connectivity.

Confidential Documents

All data is treated as local and private. At this time, there is no separate marking for "confidential" documents. However, no document leaves the user's device unless explicitly shared.

Compliance with Privacy Laws

Although fyle operates entirely on-device and does not collect personal information, we strive to align our practices with the principles of Apple App Store Review Guidelines.

Since we do not collect or transmit data externally, user rights to access, deletion, and control are fulfilled entirely within the app interface.

App Store Disclosure

In accordance with Apple's App Store Privacy Guidelines, fyle declares that no data is collected, linked to the user, or used for tracking, analytics, or advertising purposes. All required disclosures in our App Store listing accurately reflect our commitment to user privacy and local data processing.

Changes to This Policy

We may update this Privacy Policy from time to time to reflect changes in the App or applicable regulations. Users are encouraged to review the policy periodically. Any material changes will be communicated within the App interface.
